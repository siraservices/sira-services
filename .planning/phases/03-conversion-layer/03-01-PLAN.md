**Zod schema** (top of file):
- `name`: `z.string().min(1, "Name is required")`
- `email`: `z.string().min(1, "Email is required").email("Please enter a valid email address")`
- `company`: `z.string().optional()`
- `message`: `z.string().min(1, "Please describe your project")`
- Infer `LeadFormValues` type from the schema

**Booking URL constant:**
- `const BOOKING_URL = "#"` at the top of the file (placeholder — will be replaced with Google Booking Page URL before ship; noted in STATE.md blockers)

**Component state:**
- `const submitLead = useMutation(api.leads.submit)` — import api from `"../../../../convex/_generated/api"` (relative path required since tsconfig `@/*` only maps to `./src/*` and convex/ is outside src)
- `const [submitted, setSubmitted] = useState(false)`
- `const [error, setError] = useState<string | null>(null)`
- `const [loading, setLoading] = useState(false)`
- `const { register, handleSubmit, formState: { errors }, reset } = useForm<LeadFormValues>({ resolver: zodResolver(leadFormSchema) })`

**onSubmit handler:**
- Set loading true, clear error
- `await submitLead({ name, email, company: company || undefined, message, source: "homepage" })`
- On success: `reset()`, `setSubmitted(true)`
- On error: `setError("Something went wrong. Please try again.")`
- Finally: `setLoading(false)`

**Layout structure (outermost):**
- `<section id="booking" className="py-24 px-6 bg-surface">` with `max-w-6xl mx-auto` inner container
- Centered section header: heading (h2) + supporting subtext paragraph (Claude's discretion on copy)
- Two-column grid: `grid grid-cols-1 md:grid-cols-2 gap-8 items-start`

**Left column — Booking CTA card:**
- Card with darker bg or subtle orange accent border (e.g., `bg-surface-alt border border-primary/30`)
- Headline: "Ready to get started?" (or similar)
- 1-2 lines subtext about free consultation
- Orange filled CTA button: `<a href={BOOKING_URL} target="_blank" rel="noopener noreferrer">` with `bg-cta text-cta-text` classes, `hover:-translate-y-1 hover:shadow-cta-glow transition-all`
- Use Calendar or ArrowRight icon from lucide-react on the button

**"Or" divider between columns:**
- On mobile (`md:hidden`): horizontal divider with centered "or" text flanked by lines
- On desktop (`hidden md:flex`): vertical divider between columns — use absolute positioning or a flex-col element with lines flanking "or" text
- Text styling: `text-text-muted text-sm uppercase tracking-wider`

**Right column — Lead form card:**
- Card with matching styling (same bg as booking card but no accent border — visually secondary)
- If `submitted` is true: render success state instead of form — `CheckCircle` icon (lucide-react) + "Thanks! We'll be in touch within 24-48 hours." — no resubmit option
- Form fields:
  - Name + Email: side-by-side on desktop (`grid grid-cols-1 sm:grid-cols-2 gap-4`), full-width stacked on mobile
  - Company: full-width, with "(optional)" hint in label or placeholder
  - "Tell us about your project": full-width textarea
- Each input: dark-themed (`bg-surface-alt border border-text-muted/20 rounded-lg text-text placeholder:text-text-muted focus:ring-2 focus:ring-primary/40 focus:border-primary/60`), min height 44px for touch targets
- Inline errors: `{errors.fieldName && <p className="mt-1 text-sm text-red-400">{errors.fieldName.message}</p>}`
- `aria-invalid={!!errors.fieldName}` on inputs with errors, `border-red-500` conditional class
- Error state: inline message above submit button if mutation fails
- Submit button: ghost/outline style — `border border-text-muted/40 text-text bg-transparent`, disabled when loading, shows "Sending..." when loading
- `disabled:opacity-50 disabled:cursor-not-allowed` on submit button

**Responsive (DSGN-02, DSGN-03):**
- All inputs full-width on mobile (sm:grid-cols-2 only for name/email on sm+)
- Booking card renders first on mobile (natural DOM order with `order-first` if needed)
- All interactive elements have min 44px touch targets (py-3 on inputs, py-4 on buttons)
- No horizontal scroll at 375px — use px-6 section padding, max-w-6xl container
